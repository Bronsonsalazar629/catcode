# CP Sensei Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build a QLoRA-finetuned DeepSeek-Coder 6.7B competitive programming tutor trained on IOI winner data.

**Architecture:** Single-stage QLoRA finetune with 4 instruction formats (SOLVE/EXPLAIN/HINT/TEACH). Data pipeline collects from IOI repos + CodeContests + Codeforces, processes into JSONL. Training via Colab/Kaggle free tier T4. Demo app with mode selector.

**Tech Stack:** PyTorch, transformers, peft (QLoRA), bitsandbytes, datasets, gradio, BeautifulSoup (scraping)

### Task 5: Synthetic teaching data generator

**Files:**
- Create: `cp-tutor/data/generate_teaching.py`

**Step 1:** Write script that:
- Reads raw solution data from Tasks 2-4
- Generates instruction/output pairs in 4 formats using templates:
  - `[SOLVE]`: problem → clean C++ solution
  - `[EXPLAIN]`: problem → progressive walkthrough (analogy → brute force → insight → optimal)
  - `[HINT]`: problem → Socratic hints without solution
  - `[TEACH]`: algorithm tag → concept lesson with analogies
- Templates are structured prompts that can optionally be enhanced with an LLM API
- Outputs `data/raw/teaching_synthetic.jsonl`

**Step 2:** Commit

---

### Task 6: Dataset builder — merge, balance, output

**Files:**
- Create: `cp-tutor/data/build_dataset.py`

**Step 1:** Write script that:
- Reads all raw JSONL files from Tasks 2-5
- Deduplicates by problem content hash
- Balances distribution: ~30% HINT, ~30% EXPLAIN, ~25% TEACH, ~15% SOLVE
- Shuffles and splits: 95% train, 5% eval holdout
- Outputs `cp-tutor/data/train.jsonl` and `cp-tutor/data/eval.jsonl`
- Prints dataset statistics

**Step 2:** Commit

### Task 7: QLoRA training script

**Files:**
- Create: `cp-tutor/training/train_qlora.py`
- Create: `cp-tutor/training/qlora_config.json`

**Step 1:** Write `qlora_config.json`:
```json
{
  "model_name": "deepseek-ai/deepseek-coder-6.7b-instruct",
  "lora_r": 64,
  "lora_alpha": 128,
  "lora_dropout": 0.05,
  "target_modules": ["q_proj", "k_proj", "v_proj", "o_proj", "gate_proj", "up_proj", "down_proj"],
  "quant_bits": 4,
  "max_seq_length": 2048,
  "per_device_batch_size": 1,
  "gradient_accumulation_steps": 16,
  "learning_rate": 2e-4,
  "num_epochs": 3,
  "warmup_ratio": 0.03,
  "lr_scheduler": "cosine",
  "output_dir": "output/cp-sensei"
}
```

**Step 2:** Write `train_qlora.py` that:
- Loads base model in 4-bit (NF4) via bitsandbytes
- Applies LoRA adapters via peft
- Uses the CP Sensei system prompt in instruction template
- Trains on `data/train.jsonl`
- Saves LoRA adapter weights (not full model)
- Logs training loss
